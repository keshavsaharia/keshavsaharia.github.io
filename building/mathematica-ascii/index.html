Generating ASCII Art in Mathematica

While working at Wolfram Research, I did several projects with Mathematica's image processing functionality. I also did a couple fun side projects like finding Waldo in a Where's Waldo image or controlling a robotic turret to point a laser at my hand. This is another neat example, where image processing functions are used to generate ASCII art.

ASCII stands for American Standard Code for Information Interchange, and gives a standard numeric value for all characters. For example, the letter “a” has the ASCII number of 97 (capital “A” is 65). Below is a table of the full correspondence between decimal numbers and their corresponding characters.

The term “ASCII art” originates from the early Internet, where only text could be transmitted. Images were created by cleverly placing characters to create the illusion of an image (a quick Google search will yield many examples). In the spirit of Mathematica, I defined a function ASCIIart to iterate over the pixels of an image, convert each pixel to text based on intensity, and return a string representation of the result. I also defined ASCIIimage to rasterize the output of ASCIIart.

An input image is first normalized and converted to grayscale. Each pixel in the image is replaced with two characters based on its light intensity. For example, a light gray pixel is converted to a . (period and space), while a dark gray pixel is converted to a :: (two colons). After playing with the mapping from pixel intensity to text, a pretty good thresholding can be found that accurately represents the spectrum of gray values.
